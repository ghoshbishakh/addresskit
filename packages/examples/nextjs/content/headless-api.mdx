# Headless API

Use the core engine directly without React components.

## `createEngine`

Creates an engine instance from a provider. The engine caches metadata per country.

```tsx
import { createEngine } from "@addresskit/core";
import { createLibaddressinputProvider } from "@addresskit/providers-libaddressinput";

const provider = createLibaddressinputProvider();
const engine = createEngine(provider);
```

## `getSchema`

Returns the address schema for a country, including fields with labels, types, and validation rules.

```tsx
const schema = await engine.getSchema("US");
// {
//   country: "US",
//   fields: [
//     { id: "line1", label: "Street address", required: true, type: "text" },
//     { id: "line2", label: "Address line 2", required: false, type: "text" },
//     { id: "locality", label: "City", required: true, type: "text" },
//     { id: "administrativeArea", label: "State", required: true, type: "select", options: [...] },
//     { id: "postalCode", label: "ZIP code", required: true, type: "text", validation: {...} },
//   ],
//   format: "%A%n%C, %S %Z"
// }
```

## `validate`

Validates an address against its country schema. Returns structured per-field errors.

```tsx
const result = await engine.validate({
  country: "US",
  line1: "123 Main St",
  locality: "San Francisco",
  administrativeArea: "CA",
  postalCode: "94102",
});
// { valid: true, errors: [] }

const result = await engine.validate({
  country: "US",
  line1: "",
});
// { valid: false, errors: [{ field: "line1", message: "Street address is required" }] }
```

## `format`

Formats an address string according to the country's format pattern. `format`
is **async** — it loads the country metadata on demand, so you don't need to
call `getSchema`/`validate` first. The `administrativeArea` **code is resolved
to its display name** (e.g. `CA` → `California`).

```tsx
const formatted = await engine.format({
  country: "US",
  line1: "123 Main St",
  locality: "San Francisco",
  administrativeArea: "CA",
  postalCode: "94102",
});
// "123 Main St\nSAN FRANCISCO CALIFORNIA 94102"
// (US uppercases locality + administrativeArea)
```

## `clearInvalidValues`

Returns a new address object with fields cleared that are invalid for the target country. Used internally for cascade clearing on country/state change.

## Pure utilities

When you already have a `CountryAddressConfig` (e.g. from `getMetadata`), you can
skip the engine and call the underlying pure functions directly. These are the
single source of truth the engine itself uses.

```tsx
import { validateAddressConfig, formatAddress, toPostalCodeRegex } from "@addresskit/core";

const config = await provider.getMetadata("US");

validateAddressConfig(config, address); // ValidationResult
formatAddress(config, address);         // string (sync)
toPostalCodeRegex(config.postalCodePattern!).test("94102"); // boolean
```
